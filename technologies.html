Technologies Used

Python Python is primarily used to run and manage the modified Pluspy interpreter, which processes the TLA+ specifications. It coordinates the generation of random seeds and test steps, handles interactions with the nvme-cli for sending and receiving NVMe commands, and logs the test sequences and results. Python also manages retries of the test sequences to ensure reproducibility and tracks the results for debugging and bug filing purposes.
TLA+ TLA+ is used to define and formalize the behavior of the system under test, allowing for the generation of test sequences. These sequences are then executed by the modified Pluspy interpreter, which interacts with the NVMe SSD device, ensuring that the test scenarios are consistent and reproducible for failure detection and debugging.
PlusPy Pluspy is the Python interpreter for TLA+, executing the TLA+ specifications and generating test sequences based on a random seed and number of steps. It interfaces with the NVMe SSD device through blocking callbacks to nvme-cli, sending commands and receiving results. Pluspy ensures that the same test sequence can be reproduced in case of failures by sampling the same path with the same inputs, which is essential for debugging and validating potential issues.
VSCode VSCode will serve as our text editor. It is lightweight and has many beneficial plugins, including a TLA+ plugin.
Ubuntu Linux The system will run in an Ubuntu Linux enviornment.
GitHub GitHub will serve as our version control software, allowing us to easily apply updates and rollbacks.
