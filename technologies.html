= Archaeological ML Project - Technologies

Technologies Used

Python Python serves as our primary programming language, providing a robust ecosystem for scientific computing, data analysis, and machine learning. We use Python to develop our ML models, process spectroscopy data from pXRF devices, analyze geospatial information, and build the backend services that power our archaeological prediction system.
TensorFlow These machine learning libraries enable us to develop the neural networks and predictive models that integrate pXRF chemical data with spatial coordinates. They support both traditional machine learning algorithms and deep learning approaches needed for classifying pottery sherds and predicting production locations at archaeological sites.
GeoPandas These geospatial libraries extend Python's data analysis capabilities to handle geographic information. They allow us to process GPS coordinates, perform spatial joins between chemical data and location, and create visualizations that help archaeologists understand patterns of production activities at Huaca Letrada.
OpenCV This computer vision library processes drone imagery and photogrammetry data. We use OpenCV to extract features from aerial photos, prepare images for structure from motion (SfM) processing, and integrate visual data with our spectroscopic and spatial datasets in the ML prediction pipeline.
Flask These web frameworks allow us to create the APIs that connect our mobile data collection app with our ML prediction backend. They handle authentication, data validation, and serve prediction results back to field archaeologists, supporting both online and offline operations needed during fieldwork.
Flutter This cross-platform development framework enables us to build a single mobile application that works on both iOS and Android devices. The app integrates with pXRF devices, captures GPS coordinates, records field notes and images, and synchronizes with our central database when connectivity is available.
PostgreSQL with PostGIS Our database solution uses PostgreSQL with the PostGIS extension to store and query spatial data. This combination provides advanced geospatial capabilities needed to analyze relationships between chemical signatures and archaeological contexts, supporting the spatial predictions our ML tools generate.
OpenDroneMap This open-source photogrammetry toolkit processes drone imagery to create 3D models of archaeological sites using Structure from Motion (SfM) techniques. The resulting terrain models provide critical context for our chemical and spatial data analysis, helping to identify landscape features related to ancient production activities.
Visual Studio Code VSCode serves as our primary development environment. Its lightweight nature, extensive extension marketplace, and excellent integration with Python and web technologies make it ideal for our diverse development needs. Extensions for geospatial data visualization and machine learning model debugging enhance our productivity.
GitHub GitHub provides our version control system and collaboration platform. We maintain repositories for our data collection app, ML prediction tools, and documentation, allowing team members to work concurrently while tracking changes. GitHub Actions automates our testing pipeline to ensure code quality, and GitHub Pages hosts our project documentation for sharing with the archaeological community.
