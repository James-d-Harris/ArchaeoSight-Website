High-Level Requirements

Requirements Explanation
The system will randomly generate test sequences. Randomly generated tests allows SSD devices to always be tested with new sequences that are unanticipated by engineers, improving utilization and coverage.
The tests and their results will be reproducible. Reproducible results are crucial for identifying the root cause of an issue and fixing the bug.
Upon failure, the system will automatically repeat the test with the same seed and number of steps. Automation increases the number of tests that can be completed.
The system will generate logs of the test sequences and results. Logs serve as evidence for engineers when filing JIRA bugs as well as guidance on improving Western Digital's TLA+ specifications.

Our Envisioned Solution

We will build a modified version of PlusPy, a TLA+ Python interpreter, that generates test sequences for NVMe SSDs based on random seeds and a given number of steps to take. This version of PlusPy will send and receive blocking callbacks to an NVMe CLI which will issue the NVMe commands to the device and receive the results of the commands. As development progresses, we will share additional details here.
