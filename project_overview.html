Generative Testing of SSD using TLA+ Model-Simulation

Project Overview

The rapid growth of Solid-State Drives (SSDs) in consumer and enterprise applications makes their reliability critical. Traditional SSD validation relies on manually designed test plans, which fail to cover all possible failure scenarios. This results in inefficient testing, idle hardware, and undetected issues.

Our project introduces Generative Testing using TLA+, a formal specification language, combined with PlusPy, a Python-based TLA+ interpreter, to generate and execute randomized test sequences dynamically via NVMe commands. This improves test coverage, reduces bias, and ensures SSDs are tested more thoroughly.

Problem & Motivation

Current SSD validation faces major inefficiencies:

  • Test plans are limited to predefined cases, leaving unknown failure conditions untested.
  • Once test plans run out of cases, SSD hardware sits idle, wasting resources.
  • Re-executing the same test plan does not improve test coverage.

Our solution eliminates these inefficiencies by continuously generating new test sequences using TLA+ specifications, ensuring SSDs undergo unanticipated conditions, leading to better bug detection.
